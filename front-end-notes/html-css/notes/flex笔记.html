# 3.容器(父元素)属性

一共有六个属性:
	flex-direction 决定了主轴的的方向(即项目排列方向)
		flex-direction的值有:
			row:(左对齐)主轴为水平方向,起点在左端
			row-reverse:(右对齐)主轴为水平方向,起点在右端
			column:主轴为垂直方向,起点在上沿
			column-reverse:主轴为垂直方向,起点在下沿
	flex-wrap 可以让项目(子元素)换行
		flex-wrap的值有:
			nowrap:默认值, 弹性容器默认为单行,该情况下弹性子项可能会溢出容器。
			wrap:弹性容器为多行,该情况下弹性子项溢出的部分会被放置到新行，子项内部会发生断行
			wrap-reverse:沿着侧轴反方向换行
	flex-flow direction和wrap的集合 flex-flow包含了direction和wrap里面所有的值
		正常写direction和wrap需要写俩条
			比如: flex-direction:row;
				 flex-wrap:nowrap;
		使用了flex-flow写一条就可以
			比如: flex-flow:row nowarp
				 flex-flow:可以使用direction和wrap里的所有
	justify-content 定义项目(子元素)在主轴的对齐方式
		justify-content的值:
			flex-start:默认值,左对齐
			flex-end:右对齐
			center:居中对齐
			space-between:弹性项目平均分布在该行上,前后没有空间
			space-around:弹性项目平均分布在该行上，两边留有一半的间隔空间
			space-evenly:弹性项目平均分布在该行上，平均分配间隔空间,兼容性不好
	align-items 定义项目在交叉轴(纵轴)上如何对齐
		align-items的值:
			flex-start:侧轴起始位置(上,不改变主轴方向的前提)
			flex-end:侧轴结束位置(下,不改变主轴方向的前提)
			center:侧轴居中对齐
			baseline:基线对齐
			stretch:默认值,将行于行之间的高度设置为相同的值,以最长的高度为准
	align-content: 定义侧轴空白空间分布
		align-content的值和justify-content里的值一样,这个不常用

# 4.项目(子元素)属性

order 数字越大越靠后,默认为0,数字相同按照标签顺序排列

flex-grow(扩大项目) 定义项目的放大比率,默认0,只有容器有剩余空间才能扩大
	1.可以把容器的剩余空间沾满
	2.可以定义占剩余空间的份数

flex-shrink(缩小项目)定义了项目的缩小比率,默认1,如果空间不足,该项目将缩小,那个项目(子元素)设置了flex-shrink,那个项目就按照设置的值缩小,默认值1,缩小1份
	
flex-basis 和width差不多 ,定义弹性盒子元素的默认基准值

flex 是flex-grow flex-shrink flex-basis的简写 默认值为0 1 auto

align-self:用来覆盖当前弹性元素的align-items的值,相当于单独给某个项目设置align-items值 //写在想要单独改变align-items的项目上

# 5.尺寸单位

px
%:相对于父元素宽高

底下这三个在ie8及以下不兼容
vw\vh :相对于视口来说
em : 针对于父元素的font-size,
	1em相当于父元素的font-size的px
	1em=16px
rem: 针对于根元素(html)的font-size的px
	1rem相当于根元素的font-size的px
	写移动端使用rem,rem知道html的font-size就可以了,em还需要计算父元素的font-size这样每次嵌套都需要计算比较麻烦,所以使用rem比较方便
谷歌浏览器字体大小默认是:16px


物理像素:手机终端真实像素: iphone6:750x1334像素 
设备独立像素(写代码的px像素):浏览器模拟手机的像素: 375(宽)×667(高)
dpr: 设备像素比，物理像素/设备独立像素 = dpr， 一般以 Iphon6 的 dpr 为准 dpr = 
2  750/375=2,所以像素比是2

rpx:小程序独有的单位,规定任何设备屏幕下宽度为 750rpx;小程序会根据屏幕的宽度不同自动计算 rpx 值的大小
d) Iphone6 下： 1rpx = 1 物理像素 = 0.5px
