# （七）语义化的概念和好处？

# 什么是 HTML 语义化？
  • 根据页面内容的结构，选择合适的标签。如，标题用 h1-h6、段落用 p 标签等。
# 语义化的好处？
  • 在没有 CSS 的情况下，页面也能呈现出很好地内容结构。
  • 语义化使代码更具可读性，便于团队开发和维护
  • 语义化有利于用户体验（例如：title、alt、label 标签的灵活运用）
  • 语义化有利于 SEO（网页和搜索引擎建立良好沟通，有助于爬虫抓取更多的有效信息。爬虫依赖于标签来确定上下文和各个关键字的权重）

# 扩展阅读

    字符编码
    字符（Character）是各种文字和符号的总称，包括各国家文字、标点符号、图形符号、数字等。字符集（Character set）是多个字符的集合。如需正确地显示 HTML 页面，浏览器必须知道使用何种字符集。字符集种类较多，每个字符集包含的字符个数不同。
    常见字符集名称：ASCII字符集、GB2312字符集、Unicode字符集等。万维网早期使用的字符集是 ASCII。ASCII 支持 0-9 的数字，大写和小写英文字母表，以及一些特殊字符。
    GB2312是中国国家标准的简体中文字符集。它所收录的汉字已经覆盖99.75%的使用频率，基本满足了汉字的计算机处理需要。
    GBK即汉字内码扩展规范，GBK编码标准，包含全部中文字符，兼容GB2312，集简、繁体字融于一库。
    Unicode字符集为每种语言中的每个字符设定了统一并且唯一的编码，以满足跨语言、跨平台进行文本转换、处理的要求。UTF-8是Unicode的其中一个使用方式,又称万国码。用在网页上可以同一页面显示中文简体繁体及其它 语言（如英文，日文，韩文），通用性比较好。

# 外边距塌陷

# 1、父子关系边距塌陷

问题描述：

当父元素没有border/padding/overflow/float/position这些属性时，子元素设置margin-top会把父元素一块儿带下来（塌陷）

解决方法：

1、给父元素添加1px的上padding或者border, 子元素的margin-top减1px
2、给父元素添加overflow:hidden属性
# 2、相邻元素垂直外边距塌陷

问题描述：

相邻的两个标签，上一个设置了margin-bottom,下一个设置了margin-top，两个标签之间出现的外边距不是两者相加，而是取得较大的那个值

解决方法：

1、避免利用垂直外边距相加实现效果
2、给相邻的元素其中一个添加一个父元素，给这个父元素添加overflow:hidden;

# 弹性盒布局

  • 弹性盒布局是css3中的新的布局方式，把父元素设置为弹性盒容器，可以更加方便得去规定子元素的排列方式、对齐方式、剩余空间的分配。
# 一、容器属性
# 1、设置弹性容器
  • display:flex; 设置为弹性盒容器，容器本身保持块级的特点
  • display:inline-flex; 设置为弹性盒容器，容器本身保持行内的特点
# 2、主轴的方向
  • 主轴的方向决定了项目的排列方向
  • flex-direction
    • row 从左到右
    • row-reverse 从右到左
    • column 从上到下
    • column-reverse 从下到上
# 3、项目在主轴上的对齐方式（以主轴为row时为例）
  • justify-content
    • flex-start 主轴开始（左对齐）
    • flex-end 主轴结束（右对齐）
    • center 居中
    • space-around 每个项目前后空白相同（两个项目之间空白是两倍）
    • space-between 每两个项目之间的空白相同的，主轴开始和结束没有空白
    • space-evenly 主轴开始和结束以及每个项目之间的空白都相同
# 4、项目是否换行
  • flex-wrap
    • nowrap 不换行，当项目总宽度超过容器，会压缩项目
    • wrap 换行
    • wrap-reverse 换行，行倒序
# 5、项目在交叉轴上的对齐（以主轴为row时为例）
  • align-items
    • stretch 默认，拉伸，如果项目没有设置高度，把项目拉伸占满容器高度
    • flex-start 在交叉轴开始对齐 （顶部对齐）
    • flex-end 在交叉轴结束对齐 （底部对齐）
    • center 居中
    • baseline 第一行字基线对齐
# 6、多行项目对齐方式
  • align-content
    • stretch 默认，拉伸，平均分容器高度，项目如果不设置高度自动拉伸占满
    • flex-start 多行顶部对齐
    • flex-end 多行底部对齐
    • center 多行居中
    • space-around 每行前后空白相同（两个行之间空白是两倍）
    • space-between 每两行之间的空白相同的，交叉轴开始和结束没有空白
    • space-evenly 交叉轴开始和结束以及每行之间的空白都相同
# 二、项目的属性
# 1、项目排序

order : 0; 数字越大越靠后，默认都是0 ，数字相同，按照标签顺序排列

# 2、项目放大比例

flex-grow : 0 ; 默认都是0表示不放大，1表示放大，数字越大放大的比例越大

# 3、项目缩小比例

flex-shrink:1; 默认都是1，表示当容器大小不够，所有项目等比例压缩， 如果设置为0表示不压缩，数字越大压缩的越多

# 4、单个项目在交叉轴上的对齐方式

align-self： - stretch 默认，拉伸，如果项目没有设置高度，把项目拉伸占满容器高度 - flex-start 在交叉轴开始对齐 （顶部对齐） - flex-end 在交叉轴结束对齐 （底部对齐） - center 居中 - baseline 第一行字基线对齐

  • maximum-scale=1.0 ：最大缩放比例 1
  • user-scalable=no ：用户不可缩放
